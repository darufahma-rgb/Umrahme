# PROMPT FITUR: Upgrade Peta Lokasi — Data Lengkap + Integrasi Google Maps + Detail Bersejarah

**Konteks:** project Umrahme (React + TypeScript + Vite + Tailwind, react-router-dom, multi-tenant white-label). Fitur "Peta Lokasi" sudah ada (`src/pages/Peta.tsx`, `src/pages/LokasiDetail.tsx`, `src/data/lokasi.ts`, type `Lokasi` di `src/types/index.ts`). Saat ini data tipis: cuma `sejarah` 1 paragraf, ada placeholder "Peta interaktif — segera hadir", dan tidak ada koordinat / link Google Maps.

**Tujuan:** rombak total jadi konten ziarah yang kaya & akurat. Tiap lokasi punya koordinat, deep-link Google Maps yang bisa diklik, sejarah panjang berstruktur, tokoh bersejarah, keutamaan, ayat/hadits terkait, adab kunjungan, dan fakta menarik. JANGAN ubah design system (warna, font, token Tailwind seperti `text-ink`, `bg-surface-card`, `text-gold`, `border-hairline`, dll) — tetap konsisten dengan halaman lain. JANGAN sentuh fitur lain.

---

## BAGIAN 1 — Update type `Lokasi`

Di `src/types/index.ts`, ganti interface `Lokasi` jadi lengkap (tambah field baru, semua opsional kecuali yang lama tetap wajib, supaya backward-compatible):

```ts
export type TipeLokasi = 'masjid' | 'sejarah';

/** Satu sub-bagian sejarah berstruktur (heading + paragraf). */
export interface SejarahSeksi {
  judul: string;
  isi: string; // boleh beberapa kalimat
}

/** Tokoh penting yang terkait dengan lokasi. */
export interface TokohLokasi {
  nama: string;
  peran: string; // contoh: "Paman Nabi ﷺ, syahid di Perang Uhud"
}

/** Dalil (ayat atau hadits) yang relevan. */
export interface DalilLokasi {
  sumber: string;      // contoh: "QS. At-Taubah: 108" atau "HR. Bukhari"
  teks: string;        // terjemahan ringkas dalam bahasa Indonesia
}

export interface Lokasi {
  id: string;
  tipe: TipeLokasi;
  nama: string;
  namaArab?: string;   // contoh: "المسجد الحرام"
  kota: string;
  jarakKm: number;     // tetap: jarak representatif dari area hotel
  jamKunjungan: string;
  ringkas: string;     // 1 kalimat untuk list
  /** Koordinat asli untuk deep-link Google Maps. WAJIB diisi. */
  koordinat: { lat: number; lng: number };
  /** Sejarah panjang dipecah jadi beberapa seksi berheading. */
  sejarahSeksi?: SejarahSeksi[];
  /** Tetap dipertahankan sebagai fallback / ringkasan sejarah. */
  sejarah: string;
  tokoh?: TokohLokasi[];
  keutamaan?: string[];   // bullet poin keutamaan / fakta menarik
  dalil?: DalilLokasi[];
  adab?: string[];        // adab & tips kunjungan
  /** true kalau ada riwayat lemah / perlu di-flag, ditampilkan sebagai catatan. */
  catatan?: string;
}
```

export function lokasiByTipe(tipe: Lokasi['tipe']): Lokasi[] {
  return daftarLokasi.filter((l) => l.tipe === tipe);
}

export function lokasiById(id: string): Lokasi | undefined {
  return daftarLokasi.find((l) => l.id === id);
}

/** Bangun deep-link Google Maps dari koordinat (membuka app Maps di HP, web di desktop). */
export function gmapsUrl(l: Lokasi): string {
  const q = encodeURIComponent(`${l.nama} ${l.kota}`);
  return `https://www.google.com/maps/search/?api=1&query=${l.koordinat.lat},${l.koordinat.lng}(${q})`;
}

Ganti placeholder peta jadi **kartu peta yang bisa diklik membuka Google Maps**, dan render semua konten baru. Tetap pakai `PageHeader`, `EmptyState`, dan token warna yang ada. Pertahankan struktur, hanya perkaya isinya.

Aturan tampilan:
1. **Kartu Peta (klik → Google Maps):** ganti placeholder lama. Buat kartu yang ketika diklik membuka `gmapsUrl(lokasi)` di tab baru (`<a href={...} target="_blank" rel="noopener noreferrer">`). Tetap pakai motif garis diagonal yang ada sebagai latar, tapi tambah label jelas: ikon pin + teks **"Lihat di Google Maps"** dan baris kecil `{kota} · {jarakKm} km dari hotel · koordinat {lat}, {lng}`. Beri efek `active:scale-[0.99]` dan `hover:shadow-drop-soft` seperti kartu lain.
2. **Header info:** tampilkan `namaArab` (jika ada) di bawah judul dengan font serif/`text-gold` ukuran kecil, rata kanan atau di bawah eyebrow.
3. **Metadata grid** (Jarak / Kunjungan) — pertahankan seperti sekarang.
4. **Keutamaan** (jika `keutamaan` ada): kartu dengan heading "Keutamaan & Fakta" berisi list bullet bergaya (pakai titik `text-gold` sebagai marker, bukan bullet default).
5. **Sejarah:** jika `sejarahSeksi` ada, render tiap seksi sebagai sub-bagian (heading `text-ink font-semibold` + paragraf `text-body leading-relaxed`), dipisah jarak. Jika tidak ada, fallback ke paragraf `sejarah` lama. Beri heading utama "Sejarah".
6. **Tokoh Bersejarah** (jika `tokoh` ada): kartu "Tokoh Bersejarah" — list nama (`font-semibold text-ink`) + peran (`text-charcoal text-sm`).
7. **Dalil** (jika `dalil` ada): kartu "Dalil Terkait" — tiap item tampilkan teks terjemahan (italic, `text-body`) lalu sumber (`text-gold font-mono text-xs`, mis. "— QS. At-Taubah: 108").
8. **Adab Kunjungan** (jika `adab` ada): kartu "Adab & Tips" dengan list bullet.
9. **Catatan** (jika `catatan` ada): tampilkan sebagai callout lembut di bagian bawah — kotak `border border-hairline bg-surface-bone` dengan ikon kecil dan teks `text-charcoal text-sm`, diawali label **"Catatan:"** `text-gold`. Ini penting untuk meluruskan riwayat lemah (mis. Jabal Rahmah).

Semua kartu pakai pola yang sudah ada: `rounded-md border border-hairline bg-surface-card px-5 py-5 shadow-drop-card`, dengan jarak antar kartu `space-y-4` atau `mb-4`. Jaga agar mobile-first, padding `px-5`, dan ada padding bawah cukup agar konten tidak tertutup bottom-nav (`pb-24` atau sesuai pola halaman lain).

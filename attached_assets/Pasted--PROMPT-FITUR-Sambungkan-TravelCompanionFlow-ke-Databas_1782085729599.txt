## 1. Tambah kolom operasional ke tabel `tenants`

```sql
alter table tenants
  add column hotel_makkah text,
  add column hotel_madinah text,
  add column meeting_point text,
  add column guide_name text,
  add column guide_whatsapp text,
  add column tour_leader_name text,
  add column tour_leader_whatsapp text,
  add column emergency_note text;
```

Semua NULLABLE — tenant yang sudah ada tidak boleh rusak, kolom kosong berarti admin belum mengisi (lihat poin 4 untuk cara fallback-nya di sisi tampilan).

## 2. Tabel baru `travel_announcements`

```sql
create table travel_announcements (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  label text not null default 'Info',
  title text not null,
  content text not null,
  important boolean not null default false,
  published_at timestamptz not null default now()
);

create index idx_travel_announcements_tenant ON travel_announcements (tenant_id, published_at desc);
```

### RLS untuk `travel_announcements`
- AKTIFKAN RLS.
- Policy READ (`select`): publik (`using (true)`) — sama seperti `tenants` dan `agenda_items`, karena dibaca app jamaah yang tidak punya sesi Supabase Auth.
- Policy WRITE: TIDAK ada policy publik, hanya lewat admin panel dengan sesi Supabase Auth admin.

## 4. Ganti `travelCompanion.ts` dari dummy jadi query database

Di `src/data/travelCompanion.ts`:
- `getOperationalInfo(tenant: TenantRow | null)`: ubah signature agar menerima objek `tenant` (bukan `tenantId` string), kembalikan field operasional dari `tenant` jika ada, fallback ke `DEFAULT_OPERATIONAL_INFO` yang sudah ada PER FIELD jika field tersebut null di database (mis. jika admin baru isi `hotel_makkah` tapi belum isi `guide_name`, tetap tampilkan default untuk yang kosong, bukan kosongkan semuanya). Hapus fungsi ini dari yang menerima `_tenantId` yang diabaikan.
- `getLatestAnnouncement` dan `getAllAnnouncements`: ubah jadi ASYNC, query tabel `travel_announcements` berdasarkan `tenant_id`, urut `published_at` descending. `getLatestAnnouncement` ambil baris pertama saja. Jika tidak ada baris sama sekali untuk tenant tersebut, kembalikan `null` (untuk `getLatestAnnouncement`) atau array kosong (untuk `getAllAnnouncements`) — JANGAN fallback ke `DEFAULT_ANNOUNCEMENTS` dummy, karena pengumuman itu konten yang harus benar-benar berasal dari travel tersebut, kosong itu kondisi valid (lebih baik daripada menampilkan pengumuman travel lain).
- Komponen pemanggil (`TripIdentityCard`, `EmergencyGuideCard`, `TravelAnnouncementCard`, `Pengumuman.tsx`) yang sekarang memanggil fungsi-fungsi ini secara sinkron perlu disesuaikan untuk menangani versi async (gunakan `useEffect` + `useState` untuk fetch saat mount, mengikuti pola yang sudah dipakai di halaman lain seperti `AgendaLengkap.tsx`).

## 5. Tampilkan state kosong yang jelas (bukan dummy)

Ini berbeda dari pengumuman (poin 4) karena kartu kontak darurat lebih baik menampilkan placeholder yang jelas daripada hilang total (kartu ini penting untuk keselamatan).

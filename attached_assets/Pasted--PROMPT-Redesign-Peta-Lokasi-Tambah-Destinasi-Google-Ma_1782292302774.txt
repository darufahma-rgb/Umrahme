## BAGIAN B — REDESIGN UI LIST di `Peta.tsx`

Pertahankan design system yang ada (warna gold, font-mono eyebrow, hairline border, surface-card). Tapi perkaya tampilan kartu:

### B1. Tambah thumbnail / placeholder gambar di tiap kartu
Tiap kartu lokasi punya area gambar di kiri (mobile) atau atas (desktop). Jika `l.gambar` ada, tampilkan `<img>`. Jika kosong (`''`), tampilkan placeholder elegan: kotak dengan ikon lokasi + pattern garis diagonal halus (gold/primary opacity rendah), JANGAN kotak abu polos.

Contoh placeholder (mobile, di kiri kartu):
```jsx
<div className="relative h-16 w-16 flex-none overflow-hidden rounded-md border border-hairline bg-surface-bone">
  {l.gambar ? (
    <img src={l.gambar} alt={l.nama} className="h-full w-full object-cover" />
  ) : (
    <>
      <div className="absolute inset-0 opacity-[0.18]" style={{ backgroundImage: 'repeating-linear-gradient(45deg, #d4a24e 0 1px, transparent 1px 8px)' }} aria-hidden />
      <div className="absolute inset-0 flex items-center justify-center">
        <IconPeta className="h-5 w-5 text-gold/60" />
      </div>
    </>
  )}
</div>
```

### B2. Tambah badge kota berwarna
Beri badge kecil membedakan Makkah vs Madinah — misal Makkah pakai warna gold, Madinah pakai warna primary/hijau. Badge di pojok atau dekat nama.

### B3. Tampilkan jumlah lokasi per tab
Di atas list, tampilkan teks kecil: "{jumlah} lokasi" untuk tab aktif, biar terasa kaya konten.

### B4. Tingkatkan keterbacaan kartu mobile
- Beri sedikit lebih banyak ruang: nama (15px semibold), arab (gold, rtl), ringkas (2 baris, line-clamp-2), lalu baris meta (kota · jarak km · jam kunjungan kalau ada).
- Tambah micro-interaction: `active:scale-[0.99]` + `hover:shadow-drop-soft` (sudah ada, pertahankan).

### B5. Urutkan lokasi secara logis
Urutkan dalam tiap tab: lokasi utama dulu (Masjidil Haram, Masjid Nabawi), lalu berdasarkan kota (Makkah dikelompokkan, lalu Madinah). Gunakan urutan array atau sort sederhana.

## BAGIAN C — REDESIGN HALAMAN DETAIL di `LokasiDetail.tsx`

### C1. PASTIKAN tombol Google Maps berfungsi untuk SEMUA lokasi
Pakai koordinat `lokasi.koordinat` untuk generate URL. Tambahkan fungsi helper:
```ts
function gmapsUrl(lat: number, lng: number, nama: string) {
  return `https://www.google.com/maps/search/?api=1&query=${lat},${lng}(${encodeURIComponent(nama)})`;
}
```
Tombol "Buka di Google Maps" yang prominent (warna primary, ikon pin), buka di tab baru (`target="_blank" rel="noopener noreferrer"`). Tampilkan juga koordinat kecil di bawahnya (mis. "21.4225° N, 39.8262° E").

### C2. Hero detail dengan gambar / placeholder
Bagian atas: area gambar besar (16:9 atau 4:3). Jika `gambar` kosong, tampilkan placeholder elegan (pattern + ikon + nama lokasi besar di tengah), bukan kotak kosong.

### C3. Susun konten detail terstruktur dengan section jelas
Urutan tampilan:
1. **Hero** — gambar/placeholder + nama + namaArab + badge kota + jarak + jam kunjungan
2. **Tombol Google Maps** (prominent, langsung di bawah hero)
3. **Ringkasan** — `ringkas`
4. **Sejarah** — `sejarah` (paragraf pembuka) lalu tiap `sejarahSeksi` sebagai sub-bagian (judul + isi), pakai gaya "archive card" dengan nomor atau garis aksen
5. **Tokoh Terkait** — list `tokoh` (nama + peran), tampilkan sebagai kartu kecil
6. **Keutamaan** — list `keutamaan` dengan ikon bintang/check
7. **Dalil** — tiap `dalil` (sumber + teks) dalam kartu bergaya kutipan (border kiri gold, italic, sumber sebagai caption)
8. **Adab Ziarah** — list `adab` dengan ikon
9. **Catatan** (jika ada `catatan`) — kotak peringatan halus (kuning lembut) untuk klarifikasi fikih, mis. "ziarah ini opsional/sunnah"

### C4. Tampilkan catatan/disclaimer untuk lokasi tertentu
Untuk lokasi yang punya field `catatan`, tampilkan di kotak khusus bergaya "info" (bukan error). Ini penting untuk meluruskan pemahaman (mis. napak tilas vs ritual).

### C5. Navigasi antar lokasi (opsional, kalau mudah)
Di bawah, tombol "Lokasi sebelumnya / berikutnya" dalam tipe yang sama, biar jamaah bisa jelajah berurutan.

## STRUKTUR BARU BERANDA MOBILE (dari atas ke bawah)

### 1. Hero — TETAP seperti sekarang
Foto masjid, sapaan nama jamaah, logo tenant. Tidak diubah.

### 2. Kartu Fokus Utama (BARU — gantikan posisi "Kartu Fase" lama, naik ke urutan PALING ATAS setelah hero, sebelum chip akses cepat)

Satu kartu besar, isinya BERBEDA tergantung kondisi (urutan prioritas dari atas — pakai kondisi PERTAMA yang cocok):

**a. Jika `tenant.tanggal_keberangkatan` ada DAN selisih hari ke tanggal tersebut antara 1-30 hari (belum berangkat):**
- Tampilkan angka besar "H-{n}" sebagai elemen visual utama kartu (font besar, mirip ukuran angka jam di JadwalSholat).
- Di bawahnya: "Menuju keberangkatan bersama {nama_travel}".
- CTA: ke checklist persiapan (`/profil/persiapan`), karena ini fase paling relevan untuk mempersiapkan diri.

**b. Jika fase jamaah adalah `tanah-suci` (sudah berangkat) DAN ada agenda untuk hari ini (`agenda_items` dengan `tanggal` = hari ini, milik tenant aktif):**
- Tampilkan label "Agenda Hari Ini" + tanggal.
- List 2-3 item agenda terdekat (jam + judul + lokasi jika ada), diurutkan berdasarkan `jam_mulai`.
- Jika ada lebih dari 3 item, tampilkan "+{n} agenda lainnya" sebagai link ke halaman agenda lengkap (lihat poin 5 di bawah, perlu dibuat).
- CTA: "Lihat Semua Agenda" jika ada agenda, ATAU jika TIDAK ADA agenda hari ini, fallback ke CTA Ritual Navigator yang sudah ada ("Lanjutkan Panduan Hari Ini" ke `/ibadah/navigator`).

**c. Fallback (tidak ada tanggal keberangkatan terisi, ATAU lewat dari 30 hari, ATAU fase `selesai`):**
- Gunakan kartu fase yang SUDAH ADA sekarang (3 kotak fase + CTA sesuai fase) — JANGAN diubah, ini sudah berfungsi baik sebagai fallback ketika data agenda/tanggal belum tersedia.

Pertahankan struktur 6 chip yang sudah ada (Doa, Ihram, Tawaf, Sa'i, Peta, Sholat). Detail kecil, tidak wajib diubah signifikan — prioritaskan poin 2 dan 4.

### 4. Buat halaman baru sederhana yang menampilkan SEMUA `agenda_items` milik tenant aktif, dikelompokkan per tanggal (misalnya "Senin, 15 Maret 2026" sebagai header, lalu list item di bawahnya), urut kronologis. ### 6. Progress Persiapan & Sertifikat — TETAP seperti sekarang, posisi di bawah grid fitur (poin 4)

## PERHITUNGAN H-{n}

```ts
function hitungHariMenuju(tanggalISO: string): number {
  const target = new Date(tanggalISO + 'T00:00:00');
  const sekarang = new Date();
  sekarang.setHours(0, 0, 0, 0);
  const selisihMs = target.getTime() - sekarang.getTime();
  return Math.ceil(selisihMs / (1000 * 60 * 60 * 24));
}
```
Tampilkan kartu H-{n} HANYA jika hasilnya antara 1 dan 30 (inklusif). Jika 0 atau negatif (hari-H atau sudah lewat), JANGAN tampilkan H-{n}, fallback ke kondisi (b) atau (c) di atas, karena pada titik itu jamaah seharusnya sudah dalam fase `tanah-suci`.

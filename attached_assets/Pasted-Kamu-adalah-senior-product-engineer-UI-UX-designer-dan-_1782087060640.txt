Kamu adalah senior product engineer, UI/UX designer, dan product strategist untuk aplikasi travel umrah.

Tolong upgrade dashboard jamaah Umrahme supaya menjadi kualitas 10/10 dari sisi owner travel. Dashboard harus terasa seperti aplikasi resmi travel yang benar-benar mendampingi jamaah selama perjalanan, bukan sekadar kumpulan menu doa/manasik.

Konteks project:

* App utama ada di `artifacts/umrahme`
* Dashboard jamaah ada di `src/pages/Beranda.tsx`
* Komponen travel companion ada di `src/components/dashboard/TravelCompanionFlow.tsx`
* Data travel companion ada di `src/data/travelCompanion.ts`
* Halaman kartu jamaah ada di `src/pages/KartuJamaah.tsx`
* Halaman pengumuman ada di `src/pages/Pengumuman.tsx`
* Agenda lengkap ada di `src/pages/AgendaLengkap.tsx`
* Auth jamaah ada di `src/context/AuthContext.tsx`
* Tipe data ada di `src/types/index.ts`
* Jangan rebuild dari nol.
* Jangan hapus fitur lama.
* Jangan ubah total style visual.
* Upgrade flow, hierarchy, copywriting, dan komponen dashboard agar terasa premium, operasional, dan layak dijual ke owner travel.

==================================================
COPYWRITING STYLE
=================

Gunakan bahasa yang sederhana, hangat, dan operasional.

Contoh copy yang bagus:

* “Hari ini jamaah dimohon berkumpul sesuai arahan pembimbing.”
* “Buka kartu jamaah jika diminta petugas atau saat terpisah dari rombongan.”
* “Tetap bersama rombongan dan jangan ragu menghubungi pembimbing.”
* “Fokus hari ini: ikuti agenda, siapkan perlengkapan, dan jaga kondisi.”

Hindari copy yang terlalu generik:

* “Akses cepat”
* “Fitur lainnya”
* “Informasi”
* “Menu utama”

Ganti dengan:

* “Yang Perlu Dibuka Sekarang”
* “Arahan Hari Ini”
* “Bantuan Cepat”
* “Dokumen Perjalanan”
* “Panduan Ibadah”

==================================================
ACCEPTANCE CRITERIA
===================

Dashboard dianggap 10/10 jika:

1. Dalam 5 detik, owner travel langsung paham ini app untuk jamaah travelnya.
2. Jamaah langsung melihat nama travel dan identitas dirinya.
3. Rombongan, bus, kamar tampil di dashboard utama.
4. Agenda hari ini tampil dominan.
5. Instruksi hari ini tampil jelas: jam kumpul, titik kumpul, pakaian, barang dibawa.
6. Pengumuman penting tampil dengan label jelas.
7. Kontak muthowwif/tour leader mudah ditekan.
8. Kartu jamaah digital mudah dibuka.
9. Shortcut fitur berubah sesuai fase perjalanan.
10. Fitur lama tetap ada dan tidak rusak.
11. Dashboard tidak terasa seperti kumpulan menu.
12. Dashboard terasa seperti pusat komando perjalanan jamaah.
13. App tetap build tanpa error TypeScript.
14. Jalankan `npm run build` dan perbaiki sampai sukses.

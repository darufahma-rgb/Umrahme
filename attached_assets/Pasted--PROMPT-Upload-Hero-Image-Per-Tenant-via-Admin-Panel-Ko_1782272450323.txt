# PROMPT — Upload Hero Image Per Tenant via Admin Panel

## Konteks
Kita mau admin bisa upload gambar hero (background Beranda) per tenant.
Gambar disimpan di Supabase Storage, URL-nya disimpan di kolom baru `hero_image_url` di tabel `tenants`.
Di Beranda, kalau `hero_image_url` ada → pakai itu, kalau null → fallback ke gambar default.

---

## LANGKAH 1 — Tambah kolom di Supabase (jalankan manual di SQL Editor)

```sql
ALTER TABLE tenants ADD COLUMN IF NOT EXISTS hero_image_url TEXT;
```

> Jalankan ini di Supabase SQL Editor dulu sebelum deploy kode.

---

## LANGKAH 2 — `src/lib/supabase.ts`

### 2a. Tambah `hero_image_url` ke type `TenantRow`

```ts
export type TenantRow = {
  // ... field yang sudah ada ...
  hero_image_url: string | null;  // ← tambahkan ini
};
```

### 2b. Tambah function `uploadHeroImage` (di bawah `uploadLogo`)

```ts
// ── Hero Image Upload ──────────────────────────────────────────

export async function uploadHeroImage(file: File): Promise<string> {
  const ext = file.type === 'image/png' ? 'png' : file.type === 'image/webp' ? 'webp' : 'jpg';
  const filename = `hero/${crypto.randomUUID()}.${ext}`;
  const { data, error } = await supabase.storage
    .from('logos')
    .upload(filename, file, { upsert: false, contentType: file.type });
  if (error) throw new Error(error.message);
  const { data: { publicUrl } } = supabase.storage.from('logos').getPublicUrl(data.path);
  return publicUrl;
}
```

> Pakai bucket `logos` yang sudah ada, tapi subfolder `hero/` biar terpisah dari logo.

## LANGKAH 3 — `src/pages/admin/AdminTenantForm.tsx`

### 3a. Import `uploadHeroImage`

Tambahkan ke baris import dari supabase:
```ts
import {
  // ... import yang sudah ada ...
  uploadHeroImage as apiUploadHeroImage,
} from '@/lib/supabase';
```

### 3b. Tambah state baru (di dekat state `logoFile`, `logoPreview`, `existingLogoUrl`)

```ts
const [heroFile, setHeroFile]             = React.useState<File | null>(null);
const [heroPreview, setHeroPreview]       = React.useState<string>('');
const [existingHeroUrl, setExistingHeroUrl] = React.useState<string>('');
const heroInputRef = React.useRef<HTMLInputElement>(null);
const MAX_HERO_SIZE = 3 * 1024 * 1024; // 3MB
```

### 3c. Load `hero_image_url` saat edit tenant

Di dalam blok `useEffect` yang load data tenant (tempat `setExistingLogoUrl` dipanggil), tambahkan:
```ts
setExistingHeroUrl(t.hero_image_url ?? '');
```

### 3d. Tambah handler upload hero

```ts
function handleHeroChange(e: ChangeEvent<HTMLInputElement>) {
  const file = e.target.files?.[0];
  if (!file) return;
  if (file.size > MAX_HERO_SIZE) { setError('Ukuran hero image maksimal 3MB.'); return; }
  const allowedTypes = ['image/png', 'image/jpeg', 'image/webp'];
  if (!allowedTypes.includes(file.type)) {
    setError('Hero image harus berupa PNG, JPG, atau WEBP.');
    return;
  }
  setHeroFile(file);
  const reader = new FileReader();
  reader.onload = ev => setHeroPreview(ev.target?.result as string);
  reader.readAsDataURL(file);
}
```

### 3e. Upload di `handleSubmit`

Di dalam `handleSubmit`, setelah baris `let logoUrl = existingLogoUrl;`, tambahkan:
```ts
let heroImageUrl = existingHeroUrl;
if (heroFile) heroImageUrl = await apiUploadHeroImage(heroFile);
```

Lalu tambahkan `hero_image_url` ke `payload`:
```ts
const payload = {
  // ... field yang sudah ada ...
  hero_image_url: heroImageUrl || null,
};
```

### 3f. Tambah UI upload di form

Cari section upload logo di JSX (ada `fileInputRef`, tombol "Upload Logo"). Tambahkan section baru **di bawahnya** dengan desain yang sama:

```tsx
{/* ── Hero Image ─────────────────────────────── */}
<div>
  <label className="block text-[11px] font-semibold uppercase tracking-widest text-slate-400 mb-2">
    Hero Image (Background Beranda)
  </label>
  
  {/* Preview */}
  {(heroPreview || existingHeroUrl) && (
    <div className="mb-3 rounded-xl overflow-hidden" style={{ height: '120px' }}>
      <img
        src={heroPreview || existingHeroUrl}
        alt="Hero preview"
        className="w-full h-full object-cover"
      />
    </div>
  )}

  <div className="flex items-center gap-3">
    <button
      type="button"
      onClick={() => heroInputRef.current?.click()}
      className="flex items-center gap-2 px-4 py-2.5 text-[12px] font-semibold rounded-xl transition-all duration-150"
      style={{ background: 'rgba(67,56,202,0.07)', color: '#4338ca', border: '1px solid rgba(67,56,202,0.15)' }}
    >
      <svg width="14" height="14" fill="none" stroke="currentColor" strokeWidth="2" viewBox="0 0 24 24">
        <path d="M21 15v4a2 2 0 0 1-2 2H5a2 2 0 0 1-2-2v-4"/>
        <polyline points="17 8 12 3 7 8"/>
        <line x1="12" y1="3" x2="12" y2="15"/>
      </svg>
      {existingHeroUrl || heroPreview ? 'Ganti Hero Image' : 'Upload Hero Image'}
    </button>
    {(heroPreview || existingHeroUrl) && (
      <button
        type="button"
        onClick={() => { setHeroFile(null); setHeroPreview(''); setExistingHeroUrl(''); }}
        className="text-[11px] text-red-400 hover:text-red-600 transition-colors"
      >
        Hapus
      </button>
    )}
  </div>
  
  <input
    ref={heroInputRef}
    type="file"
    accept="image/png,image/jpeg,image/webp"
    onChange={handleHeroChange}
    className="hidden"
  />
  <p className="mt-1.5 text-[10px] text-slate-400">
    Maks 3MB · PNG, JPG, WEBP · Rekomendasi: 1200×600px landscape
  </p>
</div>
```

## LANGKAH 4 — `src/pages/Beranda.tsx`

Import gambar default di atas tetap ada (sebagai fallback):
```ts
import heroBg from '@assets/image_1782030121542.png';
```

Di JSX hero mobile, ganti src img dari:
```tsx
<img src={heroBg} alt="" aria-hidden
  className="absolute inset-0 h-full w-full object-cover"
  style={{ objectPosition: 'center 38%' }} />
```

Jadi:
```tsx
<img src={tenant?.hero_image_url || heroBg} alt="" aria-hidden
  className="absolute inset-0 h-full w-full object-cover"
  style={{ objectPosition: 'center 38%' }} />
```

Lakukan hal yang sama untuk hero desktop (sekitar baris 482) jika ada.

---

## Catatan
- Bucket `logos` di Supabase Storage sudah ada dan sudah public — tidak perlu buat bucket baru
- Subfolder `hero/` akan terbentuk otomatis saat file pertama diupload
- Kalau `hero_image_url` null → otomatis fallback ke `heroBg` (gambar default bawaan)
- Pastikan kolom SQL sudah ditambah dulu sebelum deploy

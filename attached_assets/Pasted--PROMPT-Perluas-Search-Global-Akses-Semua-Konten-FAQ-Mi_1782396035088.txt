## LANGKAH 2 — Tingkatkan kualitas pencarian (opsional tapi disarankan)

Di fungsi `searchAll`, tingkatkan agar mendukung pencarian multi-kata (semua kata harus cocok, urutan bebas) supaya hasil lebih relevan:

```ts
export function searchAll(query: string, limit = 10): SearchItem[] {
  const q = query.trim().toLowerCase();
  if (!q) return [];
  const kata = q.split(/\s+/).filter(Boolean);

  const scored = searchIndex
    .map((item) => {
      const kw = item.keywords ?? '';
      const judul = item.judul.toLowerCase();
      // semua kata harus muncul di judul atau keywords
      const semuaCocok = kata.every((k) => judul.includes(k) || kw.includes(k));
      if (!semuaCocok) return null;
      // skor: judul match penuh > sebagian kata di judul > hanya keywords
      let score = 0;
      if (judul.includes(q)) score = 3;
      else if (kata.some((k) => judul.includes(k))) score = 2;
      else score = 1;
      return { item, score };
    })
    .filter(Boolean) as { item: SearchItem; score: number }[];

  scored.sort((a, b) => b.score - a.score);
  return scored.slice(0, limit).map((x) => x.item);
}
```

## LANGKAH 3 — Dukungan deep-link (anchor) agar search bisa "masuk ke dalam halaman"

Agar hasil search yang mengarah ke `#id` benar-benar membuka & menyorot item di dalam halaman:

### 3a. Di `FaqFikih.tsx`
- Beri tiap item accordion atribut `id`: pada wrapper `FaqAccordion`, tambahkan `id={item.id}`.
- Tambahkan logika auto-buka + scroll saat ada hash di URL:
```tsx
import { useEffect } from 'react';
import { useLocation } from 'react-router-dom';

// di dalam komponen FaqFikih:
const location = useLocation();
useEffect(() => {
  const hash = location.hash?.replace('#', '');
  if (!hash) return;
  // beri delay agar elemen sudah ter-render
  const t = setTimeout(() => {
    const el = document.getElementById(hash);
    if (el) {
      el.scrollIntoView({ behavior: 'smooth', block: 'center' });
      el.classList.add('ring-2', 'ring-primary/40');
      setTimeout(() => el.classList.remove('ring-2', 'ring-primary/40'), 2000);
    }
  }, 200);
  return () => clearTimeout(t);
}, [location.hash]);
```
- Pastikan `FaqAccordion` membuka jawabannya jika `id` cocok dengan hash (mis. lewat prop `defaultOpen={item.id === hashAktif}` atau buka semua saat di-target). Cara sederhana: jika hash cocok, set state open item tersebut menjadi true.

### 3b. Di `PanduanIhram.tsx`
- Beri `id` pada elemen pembungkus tiap item: niat (`id={n.id}`), larangan (`id={l.id}`), dan kartu dam (`id={d.id}`).
- Tambahkan `useEffect` scroll-to-hash yang sama seperti di atas, agar hasil search dam/niat/larangan men-scroll ke kartu yang tepat.

> Catatan: jika item berada di dalam accordion yang tertutup secara default, pastikan accordion target terbuka otomatis saat hash cocok.

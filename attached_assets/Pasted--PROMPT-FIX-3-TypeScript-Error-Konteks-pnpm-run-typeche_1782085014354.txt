## Error 1 & 2 — `src/data/searchIndex.ts`

```
src/data/searchIndex.ts(131,14): error TS2339: Property 'deskripsi' does not exist on type 'ManasikModul'.
src/data/searchIndex.ts(133,41): error TS2339: Property 'deskripsi' does not exist on type 'ManasikModul'.
```

Penyebab: kode mengakses `m.deskripsi`, tapi field yang benar pada interface `ManasikModul` (lihat `src/data/manasikInteraktif.ts`) adalah `subjudul`, bukan `deskripsi`.

Perbaikan di `src/data/searchIndex.ts`, bagian yang membangun index untuk Manasik Interaktif:
- Ganti semua `m.deskripsi` menjadi `m.subjudul`.

SEKALIAN perbaiki bug terkait yang ditemukan saat investigasi (bukan error TypeScript, tapi bug navigasi nyata): route tujuan untuk item manasik di search index saat ini `to: \`/manasik/${m.id}\`` — route ini TIDAK ADA di `App.tsx`. Route yang benar dan sudah terdaftar adalah `/panduan/manasik-interaktif/${m.id}` (lihat `App.tsx`, route `/panduan/manasik-interaktif/:modulId`). Perbaiki `to` di search index ini supaya hasil pencarian Manasik benar-benar mengarah ke halaman yang ada, bukan 404.

## Error 3 — `src/pages/Pengumuman.tsx`

**Opsi A** (jika `AnnouncementCard` memang dimaksudkan punya state collapse/expand): tambahkan prop `defaultOpen?: boolean` ke interface props `AnnouncementCard`, gunakan sebagai initial value untuk state buka/tutup di dalam komponen tersebut.

**Opsi B** (jika `AnnouncementCard` tidak punya/tidak butuh state collapse sama sekali, kontennya memang selalu tampil penuh — sesuai komentar `// Card pengumuman — konten selalu tampil` yang ada di atas komponen ini): hapus saja prop `defaultOpen={i === 0}` dari pemanggilan komponen di baris yang error, karena tidak relevan.

Periksa isi komponen `AnnouncementCard` terlebih dahulu untuk menentukan opsi mana yang sesuai — JANGAN asal pilih tanpa membaca implementasinya, karena salah pilih bisa menghilangkan fitur yang dimaksudkan atau meninggalkan kode mati.
